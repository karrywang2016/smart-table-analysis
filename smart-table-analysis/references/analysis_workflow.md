# 智能表格分析工作流程

## 完整分析流程

### Phase 1: 数据加载与探查

**触发**：用户提供 Excel/CSV 文件路径或上传文件

1. **加载文件**：使用 `data_loader.py` 的 `schema` 动作
   ```bash
   python data_loader.py <file> --action schema
   ```
   返回：文件格式、列信息、数据质量评估、前5行预览

2. **生成数据概览**：向用户呈现
   - 文件基本信息（格式、大小、行数列数）
   - 列分类（数值列 / 分类列 / 时间列）
   - 数据质量评分和问题列表
   - 数值列的基本统计

3. **数据问题预警**：
   - 高缺失率字段 → 建议填充或删除
   - 重复数据 → 建议检查
   - 异常值 → 建议排查

4. **给出下一步操作选项**（见下方「选项生成规则」），等待用户选择

5. **确认机制**：用户选择后，复述确认分析类型、目标列、分组维度、图表类型，确认无误后再执行

### 选项生成规则

根据数据特征和用户原始提问，生成 3-5 个可执行操作选项：

- **数据驱动**：选项必须基于实际存在的列和数据分布
- **意图匹配**：优先匹配用户原始提问中的关键词（"对比""趋势""排名"等）
- **多样性**：覆盖不同分析维度（分组/趋势/分布/异常/交叉）
- **具体明确**：每个选项说明"分析什么 + 用什么图表"
- **推荐标记**：最匹配用户意图的选项标记为"推荐"

### 确认机制

用户选择后，**必须复述确认**：
- 明确分析类型、目标列、分组维度、图表类型
- 例如："好的，我来按地区分组统计销售额对比，生成柱状图。开始生成..."
- 确认无误后再执行脚本

### Phase 2: 自然语言分析

**触发**：用户在 Phase 1 选项中选择一项，或主动提出分析需求

1. **确认理解**：用户选择后，**必须复述确认**：
   - 分析类型（对比/趋势/分布/构成/关系/异常/排名）
   - 目标列（哪些列参与分析）
   - 分组维度（按什么分组/拆分）
   - 图表类型（柱状图/折线图/饼图等）
   - 例如："好的，我来按「地区」分组统计「销售额」对比，生成柱状图。开始生成..."

2. **代码生成与执行**：
   - 参照 `pandas_patterns.md` 映射表生成 Pandas 代码
   - 使用 `analyzer.py` 执行代码：
     ```bash
     python analyzer.py <file> --action exec --code "<generated_code>"
     ```
   - 或使用内置分析动作（stats/pivot/correlation/groupby/compare/time/cross/outliers）

3. **结果解析**：
   - 将 JSON 结果转为结构化文字描述
   - 提取关键数字和洞察
   - 发现异常模式或显著差异

### Phase 3: 可视化生成

**触发**：分析结果需要图表展示

1. **选择图表类型**：参照 `chart_types.md` 决策树
2. **生成图表**：
   ```bash
   python chart_generator.py <file> --chart-type <type> --x-col <col> --y-col <col> --output <path>
   ```
3. **图表 + 洞察组合**：每张图表配一段文字解读

### Phase 4: 洞察提炼

**触发**：分析完成后自动提炼关键洞察

洞察格式：
```
📊 [洞察标题]
   数据支撑：[具体数字]
   业务含义：[解读]
   建议：[下一步行动]
```

洞察优先级：
- P0：强异常（数据缺失>30%、完全偏斜分布）
- P1：显著差异（组间差异>50%、强相关性>0.7）
- P2：一般发现（趋势方向、分布特征）

### Phase 5: 追问处理

**触发**：用户对分析结果追问

支持的追问类型：
| 追问示例 | 处理方式 |
|---------|---------|
| "再按地区拆一下" | 在现有分析基础上添加 groupby 维度 |
| "对比上月" | 计算环比/同比，生成对比图表 |
| "去掉XX的数据" | 添加过滤条件后重新执行 |
| "换个图表类型" | 切换 chart_type 重新生成 |
| "导出这个结果" | 调用 exporter.py 导出 |
| "再做一次XX分析" | 保留上下文，新分析起点 |
| "TOP 10 是什么" | 添加排序和截取 |
| "加一个移动平均" | 添加 rolling 计算 |
| "两个表关联一下" | merge 操作 |

追问时保持数据上下文：记录已加载的文件、已执行的列、已生成的图表，避免重复加载。

### Phase 6: 结果导出

**触发**：用户要求导出结果

支持格式：
1. **Excel**：`exporter.py --action excel`
2. **CSV**：`exporter.py --action csv`
3. **HTML报告**：`exporter.py --action html`（含图表+洞察+统计+预览）

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 文件格式不支持 | 提示支持的格式列表，建议转换 |
| 列名不存在 | 显示实际列名，让用户选择 |
| 执行代码报错 | 捕获错误信息，自动修正后重试，最多3次 |
| 内存不足 | 建议先筛选/采样再分析 |
| 中文编码问题 | 自动尝试 utf-8-sig / gbk / gb18030 |
