# 选项生成规则与确认机制

## 选项生成规则

分析完数据后，根据数据特征和用户原始提问，给出 3-5 个可执行操作选项。

### 规则

| 规则 | 说明 |
|------|------|
| **数据驱动** | 选项必须基于实际存在的列和数据分布，不能凭空捏造 |
| **意图匹配** | 优先匹配用户原始提问中的关键词（"对比""趋势""排名"等） |
| **多样性** | 覆盖不同分析维度（分组/趋势/分布/异常/交叉），不要全部是同一类型 |
| **具体明确** | 每个选项说明"分析什么 + 用什么图表"，不要只说"分析XX" |
| **推荐标记** | 最匹配用户意图的选项标记为"【推荐】" |

### 生成流程

1. **提取用户意图关键词**：从用户原始提问中提取动词（对比/分析/统计/排名/趋势等）
2. **匹配数据列**：找到与分析意图匹配的列
3. **组合选项**：生成 3-5 个不同维度的分析选项
4. **排序**：最匹配的放前面，标记"【推荐】"

### 选项模板

```
💡 基于你的数据，我建议以下分析方向：

1. 【推荐】按"XX"分组统计"YY"对比 → 柱状图
2. 按"XX"查看"YY"趋势 → 折线图
3. 按"XX"统计"YY"占比 → 饼图
4. 查看"YY"与"ZZ"的相关性 → 散点图
5. 检测"YY"异常值 → 箱线图

你想做哪个？或者有其他想法？
```

## 确认机制

用户选择选项后，**必须复述确认**才能开始生成。

### 为什么需要确认

- 数据分析耗时耗资源，错误的分析浪费用户时间
- 列名/维度/图表类型理解偏差会导致结果无用
- 确认体现了对用户需求的尊重

### 确认模板

```
好的，我来：
- 分析类型：分组对比
- 目标列：销售额
- 分组维度：地区
- 图表类型：柱状图

确认无误，开始生成...
```

### 确认内容清单

| 要素 | 必须确认 |
|------|---------|
| 分析类型 | 对比/趋势/分布/构成/关系/异常/排名 |
| 目标列 | 哪些数值列参与分析 |
| 分组维度 | 按什么分组/拆分（如有） |
| 图表类型 | 柱状图/折线图/饼图/散点图/箱线图等 |
| 输出格式 | Excel/文本/图表（根据用户需求） |

### 用户修改

如果用户说"不对，我想按XX改一下"，则：
1. 记录修改
2. 重新确认
3. 确认后再执行
